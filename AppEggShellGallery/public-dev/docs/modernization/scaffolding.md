# Scaffolding (Goal B)

`eggshell create-app` is the front door for every new EggShell project. It is currently broken: a
freshly scaffolded app does not run `eggshell dev-web` cleanly.

---

## Why it is broken: the drift table

`Meta/LibScaffolding` (TypeScript + `templates/`) backs the `create-*` CLI commands. It is a snapshot
of an older framework state and has drifted from the known-good reference apps in several concrete ways:

| Area | Template generates | Current working apps expect | Effect |
|------|--------------------|-----------------------------|--------|
| `eggshell.json` | flat `renderDependencies` key | nested `render.{dependenciesToRtCompile, additionalModulesToOpen, componentLibraryAliases, componentLibraryPaths, componentAliases}` | Render compile misconfigured or fails |
| `App.fsproj` | hardcoded `.fs` includes, no `_autogenerated_` entries, no `<Content Include="**/*.render"/>` | each component pairs source with `_autogenerated_/.../*.TypeExtensions.fs` + `*.Render.fs` | F# build fails on unresolved generated modules |
| `package.json` | RN/mobile-centric, no webpack / dev-web `start` script, hardcoded `file:` paths to libs that will not exist | a `webpack-dev-server` start path | `dev-web` cannot start |
| Components / routes / dialogs | still emit `.render` files (deprecated DSL) | pure F# is the modern path | perpetuates the DSL you want to retire |
| Boilerplate | company-specific libs (`LibUiChaldalAuth`, `LibUiChaldal`, hardcoded `Landing/Bananas/Mangoes/DoSomething`) | generic minimal app | new app drags in company libs and references files that do not exist |
| `index.html` / entry | references `/bundle.js`, `.build/native/...` with no build wiring | webpack-produced bundle | runtime "cannot find module" |

**Net:** scaffolding fails at multiple independent layers (config schema, project file, web tooling,
deprecated DSL, company coupling). These must be fixed together, not one at a time.

---

## What a correct template looks like

A correct template is, by definition, the canonical modern EggShell app. It anchors Goals C and D.
The fix plan:

1. **Rewrite `eggshell.json.template`** to the current nested `render` schema (or an empty render
   block when the app is pure F# only). Remove all company-specific coupling.

2. **Rewrite `App.fsproj.template`:** minimal references (`LibClient`, `LibRouter`, `FablePlugins`),
   correct `_autogenerated_` + `Content` includes. After DSL retirement, no autogenerated render
   entries at all.

3. **Fix `package.json.template`:** add the webpack dev-server path; drop dead `file:` deps and
   mobile-only bloat; add Playwright as a dev dependency.

4. **Stop emitting `.render` from `create-component` / `create-route` / `create-dialog`.** Emit pure
   F# `[<Component>]` files instead. This ties directly into Goal A.

5. **Auto-update the `.fsproj` on `create-component`** (today it is a manual, error-prone step).
   Parse the project file, insert the new file in the correct compile order position.

6. **Add a CI smoke test:** `create-app -> dev-web` must build green. Without this, scaffolding will
   silently rot again.

7. **Add `android/` and `ios/` native project templates** (from the gallery, parameterized by app
   name) so native dev is wired out of the box.

---

## The TODO reference app as the template source

The cleanest way to fix scaffolding is not to patch the existing templates incrementally. Instead:

1. Build the concrete `SuiteTodo/` in-repo reference implementation (Phase 5, see
   [Phased Plan](./phased-plan.md)) and get it green on all platforms.
2. Templatize it into `Meta/LibScaffolding/templates`: replace domain-specific TODO content with
   generic placeholders; keep the modern conventions (nested render schema, pure F#, correct
   `App.fsproj`, generic app shell).
3. The resulting template is both the scaffold output and the living proof that conventions work.

This "concrete first, templatize second" order is important: it avoids writing templates for a
framework shape that has not been validated end to end.

## The `eggshell` CLI source

The CLI is TypeScript/Node (`Meta/AppEggshellCli`), delegating to:

- `Meta/LibEggshell`: project discovery (`eggshell.json` walk-up, project types).
- `Meta/LibScaffolding`: `create-app`, `create-component`, `create-route`, `create-dialog`,
  `rename-component`, `create-third-party-wrapper`, `convert-component`.
- `Meta/LibRtCompilerFileSystemBindings`: drives the render compiler over the filesystem, TypeExtensions
  generation, webpack/native build orchestration.
- `Meta/LibFablePlus`: runs Fable and bundling.

After DSL retirement, `LibRtCompilerFileSystemBindings` shrinks significantly (no more
`processAllRender`, no watch hooks). The `renderdsl` command and the `render` section of
`eggshell.json` are deleted.

Switching the CLI itself to .NET (from TypeScript/Node) is on the long-term roadmap but is not part
of the current initiative.
