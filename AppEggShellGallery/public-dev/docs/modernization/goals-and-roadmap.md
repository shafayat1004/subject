# Goals & Roadmap

Future work for EggShell. More of a prioritized "future work" list than a committed schedule. Items
carry a status derived from a 2026-07 codebase check. For the phase-by-phase execution plan, see
[Phased Plan](./modernization/phased-plan.md).

This page supersedes `docs/basics/roadmap.md`. All prior content has been folded in, including the
candid "Platform Fundamentals / Higher Level Abstractions / Tooling / Conveniences / Gallery
Improvements / Component Library Improvements / Unsorted TODOs" lists from that file.

---

## Short-term (current initiative, Goals A through E + I)

These ship together as the current phase. They reinforce each other and are intentionally scoped to the
now-migrated toolchain (Fable 5, .NET 10 SDK).

### A. Retire the render DSL

Convert all framework `.render` components to pure F#, then delete the DSL machinery entirely.

- Upgrade `eggshell convert-component` to write the `.fs` in place and delete the `.render` (today it
  only prints to stdout).
- Convert framework-owned `.render` files (`LibClient`, `LibUi*`, `LibRouter`, `ThirdParty`),
  reviewing the generated F# for readability.
- Once all converted: strip the render compile from the build, delete `Meta/AppRenderDslCompiler`,
  `SuiteDsl/LibDsl`, and `Meta/LibRenderDSL`, simplify `eggshell.json` / `initialize`. Deleting the
  per-file render-compiler subprocess stage (~548 spawns repo-wide) is also a build-speed win (see E).

**Status:** Done for product code. The gallery is 100% pure F#; `Lib*` and `ThirdParty` have 0
`.render` files. 41 `.render` files remain as compiler test fixtures only, all under
`Meta/AppRenderDslCompiler/compiler.Tests/tests/`. DSL machinery deletion is the remaining closure
step, deferred until the full audit pass confirms no regressions. See
[Render DSL Retirement](./modernization/render-dsl-retirement.md).

### B. Fix scaffolding

`eggshell create-app` must produce a working, modern app. A correct template is, by definition, the
canonical modern EggShell app, so this also anchors Goals C and D.

- Rewrite `eggshell.json` / `App.fsproj` / `package.json` templates to the current schema + web
  dev-server tooling.
- Stop emitting `.render` from `create-component` / `create-route` / `create-dialog`; emit pure F#.
- Decouple from company-specific libraries; ship a generic minimal app.
- Auto-update the `.fsproj` on `create-component` (today it is a manual, error-prone step).
- Add a `create-app -> dev-web` CI smoke test so scaffolding cannot silently rot again.

**Status:** Not started. See [Scaffolding](./modernization/scaffolding.md) for the detailed drift table and fix plan.

### C. Reduce component verbosity

Lean into the `[<Component>]` + hooks path. After the DSL is gone: scaffolding templates for the
common component shapes, helpers for the subscribe-render pattern, fewer mandatory `Estate` /
`Pstate` / `Actions` declarations for trivial components.

**Status:** Incremental alongside A. The `[<Component>]` path is established. Approximately 16
components still use `TapCapture` shim rather than `LC.Pressable` with explicit label/role/testId.

### D. Standardize frontend directory structure

Structure is already fairly consistent (`Components/<Name>/<Name>.fs`, `ComponentRegistration.fs`,
ordered `.fsproj`). The biggest simplification comes automatically when `_autogenerated_` is removed
after DSL retirement. Codify the remaining conventions in a written guide and in the corrected
scaffolder (B).

**Status:** Incremental alongside A/B. Convention guide exists in CLAUDE.md; formal docs page pending.

### E. Speed up the frontend build

Frontend build cost is a compound of independent stages. Near-term, independent levers:

- Fingerprint-cache the LibStandard precompile (currently reruns even when nothing changed): likely
  the biggest warm-build win.
- Re-enable parallel type-check (`--noParallelTypeCheck` is currently set); debounce the webpack
  trigger instead.
- Parallelize the dependency-tree build (currently serial `Promises.inSeries`).
- Lighten dev source maps (`cheap-module-source-map` instead of `eval-source-map`).

The structural wins come from A (deleting the 548-spawn render stage) and F (Fable 5 caching/codegen;
already done). Bundler swap (esbuild/Vite) is the large long-term win.

**Status:** Partial. The render subprocess stage is eliminated by Goal A. Quick wins not yet applied.
See [Build Performance](./modernization/build-performance.md).

### I. Frontend render hygiene: key warnings + style leaks

Two classes of dev-console warnings recur across the frontend and the gallery.

**Unique "key" prop warnings.** Caused by static `[| el1; el2; ... |]` arrays handed to a component
as children via a bare cast, so React renders a keyless list. Fix: route static child arrays through
`castAsElementAckingKeysWarning` / `tellReactArrayKeysAreOkay` (which call `React.Children.toArray`
to inject stable keys) instead of bare `castAsElement`. Long-term: have the `element` / `elements` CE
builder auto-key multi-child output structurally (Goal C, Goal A).

**Style leaks.** Caused by `makeViewStyles` / `makeTextStyles` running inside render (inline in a
`styles` array, or inside a component body / `Pointer.State` / `With.ScreenSize` callback), causing
the style system to re-allocate an uncacheable style object on every render. Fix: static styles to top-level
`let`; parametrized styles to `ViewStyles.Memoize` / `TextStyles.Memoize`, keyed on primitives, never
on whole `Theme` / `Colors` records.

**Status:** In progress. Key-warning fix applied at `AppShell/Context/Context.fs`; style-leak rule
codified in `docs/fsharp/styling.md`; targeted memoization applied to `Sidebar.Item`, `Nav.Top.Item`,
`ToggleButton`, `DateSelector`, `GalleryHeadings`, `Input.File`, `LabelledFormField`.

## Platform & backend (later, Goals F through H)

Explicitly later than the active initiative. Goal F is the baseline that unblocks the rest.

### F. Platform baseline: .NET 10 + Fable 5

.NET 7 was out of support (EOL May 2024), so this was also a security/maintenance item. Done:

- `global.json` bumped to .NET 10 SDK 10.0.301.
- Fable tool bumped to 5.4.0, `Fable.Core` to 5.0.0, `Fable.React` to 10.0.0-alpha.1.
- Custom `Meta/FablePlugins` (`[<Component>]` attribute) updated for Fable 5 plugin API.
- Native and web gallery builds green.

Many project TFMs still target older frameworks. The full repo-wide TFM bump to `net10.0` is a
separate, later step that interacts with the Orleans upgrade (see G).

**Status:** Done (toolchain migrated). Per-project TFM bump: not started.

### G. PostgreSQL + Orleans 3.7 to 7.x upgrade

One coordinated backend storage workstream. Orleans 3.7 to 7 is a **hard, non-rolling** migration
(wire protocol and grain identity changed): stand up a new cluster and decommission the old one. The
Postgres storage rewrite touches the same membership / clustering / storage seams, so both should be
done **once, together**, against the target Orleans version.

Order: decide .NET LTS + Orleans target, then abstract the connection factory and provider selection,
then port DDL / stored-procs + membership tables, then Postgres storage handlers, then run the full
simulation suite in `TestDataSeeding` mode against Postgres.

**Status:** Not started. See [Architecture: Hosting & Persistence](./architecture/backend-hosting-persistence.md)
for the concrete seam map.

### H. ReactXP to react-native-web seam swap

ReactXP is archived upstream; `@chaldal/reactxp` was the only living fork, and it could not be nudged
forward without a substantial rewrite (React 19 removed `childContextTypes`, `findDOMNode`; the New
Architecture deprecated `setNativeProps`, `findNodeHandle`). See [ReactXP to RNW](./modernization/reactxp-to-rnw.md).

The contained seam is `LibClient/src/Rn`. F# components above that seam call `Rn.*` / `LC.*`
and are insulated from the swap. The seam has been re-implemented against RN + react-native-web
(`RnPrimitives.fs`); `@chaldal/reactxp` is removed as a dependency. The stack now runs React 19.2,
react-native 0.86, react-native-web 0.21.2, RNGH 3.0.2, and Reanimated 4.5.1, with the New Architecture
(Fabric) enabled.

A real **animation story** (spring/physics motion, gesture-driven and layout/route
transitions via Reanimated 4 + RNGH 3 + Moti) is the highest-visibility frontend win, and the New
Architecture that unlocks it is now enabled.

**Status:** Done. Primitive layer migrated to react-native-web; `@chaldal/reactxp` removed; React 19 /
RN 0.86 / RNW 0.21.2 / RNGH 3 / Reanimated 4 / New Architecture all in place. AppTodo verified on
Android and iOS. Remaining: gallery and PerformancePlayground native passes, plus scaffolding polish.

### Security hardening

A focused framework auth/SQL/crypto pass. See [Security Review](./modernization/security-review.md) for the full
candidate findings table. High-priority items to verify first:

- Possible **default-allow** when a subject/view has no session handling configured
  (`maybeSessionHandling = None` mapping to `Grant`).
- Schema/lifecycle/ecosystem **names interpolated** into SQL via `sprintf` (identifiers are not
  parameterized; add a whitelist or `QUOTENAME`-style escape).

**Status:** Not started.

## Platform Fundamentals

Items from the original roadmap that remain open.

- **Packaging for mobile:** `eggshell package-android` and `eggshell package-web` exist; end-to-end
  store packaging is still evolving. (Partial.)
- **F# string interpolation adoption:** projects use `LangVersion 7.0` so interpolation is available;
  the codebase still mostly uses `sprintf` by convention. (Partial.)
- **Switch the `eggshell` toolchain to .NET:** CLI is still TypeScript/Node (`Meta/AppEggshellCli`).
  (Not done.)
- **Move Subject-related components out of `LibClient`:** `LibClient` still references
  `LibLifeCycleTypes` and Subject-oriented components; `LibLifeCycleUi` was meant for that boundary
  but was never cleanly separated. (Not done.)
- **Env-variable-based configuration:** apps still use per-environment `configSourceOverrides.*.js`
  files rather than first-class env variable support. (Not done.)
