# EggShellFmt

`eggshell-fmt` -- the EggShell F# column-alignment formatter. A repo-local dotnet tool that applies
the hand-maintained alignment rules from `AppEggShellGallery/public-dev/docs/fsharp/formatting.md`
that Fantomas cannot do.

## Install (one-time per machine, or after a version bump)

```bash
./Meta/EggShellFmt/install.sh
```

This packs the tool into the git-ignored local feed (`Meta/EggShellFmt/nupkg`, wired in
`nuget.config`) and registers it in `.config/dotnet-tools.json` -- analogous to `dotnet tool restore`,
which cannot build a tool from source.

## Use

```bash
dotnet tool run eggshell-fmt -- LibClient/src            # format files/trees in place
dotnet tool run eggshell-fmt -- path/to/File.fs          # format specific files
dotnet tool run eggshell-fmt -- --check LibClient/src    # CI: don't write, exit 3 if drift
dotnet tool run eggshell-fmt -- --quiet LibClient/src    # print only changed files
```

Skips `obj/`, `bin/`, `_autogenerated_/`, `node_modules/`, `.git/`, and `*.fs.js`.

## What it enforces

| Rule | Construct | Default |
|---|---|---|
| 2a | record TYPE fields `Name: Type` (align type after `:`) | always |
| 2b | DU cases `\| Case of ...` (align `of`) | always |
| 2c | short match arms `\| pat -> body` (align `->`, only when every arm is inline) | always |
| 7  | record LITERAL fields `Name = value` (align `=`) | always |
| 2d | `let` / `and` binding groups (align `=`) | opt-in |
| 13 | CE `let!` / `and!` bindings (align `=`) | opt-in |

Plus safe normalization: leading tabs to 4 spaces, strip trailing whitespace, CRLF to LF, single
trailing newline.

**Opt-in `=` groups:** a `let`/`and`/`let!`/`and!` group is aligned only when the author already
aligned it (at least one binding has more than one space before its `=`). Ragged groups are left
ragged, so the tool never imposes let-alignment.

## Design

Line-based: every structural scan runs on a masked copy of the line where string/char literals and
comments are blanked to spaces (length-preserving), so markers are never matched inside them and edits
apply to the real line at the same indices. Lines inside a multi-line string or block comment are left
byte-for-byte. Idempotent.

It does **not** reflow long lines, break signatures, place brackets, or fix operator spacing inside
expressions -- run Fantomas for those (`dotnet tool run fantomas <file.fs>`). The two are complementary
and both idempotent; this tool never calls Fantomas.

### Known limitations (safe fallbacks, never corruption)

- DU case with a function-typed payload (`| Fn of (int -> int)`) is left unaligned rather than risk
  the inner `->`.
- A match split by blank lines aligns each blank-separated segment independently.
- Function params / named-args (`name: Type,` / `name = value,` with trailing commas) are left
  untouched; rules 2a/7 cover records, which never use commas between fields.

Wrapped by the `fsharp-format` skill.
