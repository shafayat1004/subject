# fsharp-format

Reformat F# to the EggShell house style. The alignment rules Fantomas cannot do are handled by a
repo-local dotnet tool, **`eggshell-fmt`** (source: `Meta/EggShellFmt/`, F#, no external deps).

Canonical spec: `AppEggShellGallery/public-dev/docs/fsharp/formatting.md`.

## Run it

```bash
# format files / trees in place (skips obj/, bin/, _autogenerated_/, *.fs.js)
dotnet tool run eggshell-fmt -- LibClient/src
dotnet tool run eggshell-fmt -- path/to/File.fs another/File.fs

# CI / pre-commit: don't write, exit 3 if anything would change
dotnet tool run eggshell-fmt -- --check LibClient/src
```

First use on a machine (or after a version bump): `./Meta/EggShellFmt/install.sh` once. It packs the
tool into the git-ignored local feed and registers it in `.config/dotnet-tools.json` (like
`dotnet tool restore`). After editing `Meta/EggShellFmt/`, re-run `install.sh`.

The tool is **idempotent** (run twice = no change) and **safe** (masks strings/comments before
scanning, never edits inside them; leaves lines inside multi-line strings/block comments untouched).
Run it after editing F#, before declaring the work done.

## What it enforces (the rules Fantomas skips)

| Rule | Construct | Action | Default |
|---|---|---|---|
| 2a | record TYPE fields `Name: Type` | align the type after `:` | always |
| 2b | DU cases `\| Case of ...` | align the `of` keyword | always |
| 2c | short match arms `\| pat -> body` | align `->` (only when EVERY arm is inline) | always |
| 7  | record LITERAL fields `Name = value` | align `=` | always |
| 2d | `let` / `and` binding groups | align `=` | **opt-in** |
| 13 | CE `let!` / `and!` bindings | align `=` | **opt-in** |

Plus safe normalization: leading tabs to 4 spaces, strip trailing whitespace, CRLF to LF, single
trailing newline. Alignment style is canonical: exactly one space after the longest member.

**Opt-in for `=` binding groups (2d / 13):** a `let`/`and`/`let!`/`and!` group is aligned only when
the author ALREADY aligned it (at least one binding has >1 space before its `=`). Ragged groups are
left ragged, so the tool never imposes let-alignment you didn't ask for. To opt a group in, add
padding before one `=`; to keep it ragged, leave single spaces. Record/type/DU/match rules are
structural and always applied.

**Grouping:** a group is a contiguous run of lines at the same indent, broken by a blank line, a
differently-shaped line, or a string/comment. Insert a blank line to split an over-eager group.

## What it does NOT do (use Fantomas for these)

Alignment pass only. It does not reflow long lines, break signatures, place brackets, or fix operator
spacing inside expressions. Fantomas does those:

```bash
dotnet tool run fantomas <file.fs>     # spacing/indent/brackets/line-breaks (~60% of the spec)
```
Both are idempotent so order does not matter; Fantomas is optional (the tool never calls it). Do NOT
run Fantomas in `--check`/strict mode on aligned files: it flags the manual alignment as wrong.

## Known limitations (safe fallbacks, never corruption)

- **DU case with a function-typed payload** (`| Fn of (int -> int)`): the block is left unaligned
  rather than risk misreading the inner `->`. Align by hand if desired.
- **A match split by blank lines**: each blank-separated segment aligns independently.
- **Function params / named-args** (`name: Type,` / `name = value,` with trailing commas): left
  untouched on purpose. Rules 2a/7 cover records only; records never use commas between fields.

## The alignment rules by hand (rare cases you edit without the tool)

```fsharp
// 2a record type: type aligned          // 2b DU: of aligned
type Address = {                          | Short      of ShortStatus
    Street:      string                   | Medium     of MediumStatus
    City:        string                   | LongerCase of LongerStatus
    PostalCode:  string
    CountryCode: string                   // 2c match: -> aligned (all arms inline)
}                                         | ShortCase _    -> "short"
                                          | LongerCaseName -> "longer"
// 7 record literal: = aligned (always)
return {                                  // 2d/13 = groups: opt-in (author aligned already)
    FirstName   = firstName               let maybeCurrentStatus  = getStatus id
    PhoneNumber = phoneNumber             let maybeCurrentDefault = getDefault ()
}
```

Never align `->` when any arm breaks to its own line. A blank line ends a group.
